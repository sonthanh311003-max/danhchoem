# PROJECT CONSTITUTION
# MemoryOS Project Constitution

Version: 1.0

## Mission

MemoryOS is not a website builder.

MemoryOS is an Experience Builder.

Our mission is to help people preserve life's most meaningful moments through beautiful interactive experiences.

Every design decision must support this mission.

---

## Core Principles

1. Emotion First
2. Simplicity Over Complexity
3. Beautiful By Default
4. Mobile First
5. Accessibility First
6. Performance Matters
7. AI Assists, Humans Decide
8. Every Memory Deserves Beauty

---

## Development Rules

Before implementing any feature:

1. Think first.
2. Explain architecture.
3. Then implement.

Never directly start coding.

---

## UI Rules

Never hardcode colors.

Never hardcode spacing.

Never hardcode typography.

Always use design tokens.

---

## UX Rules

No page should feel like a dashboard.

Everything should feel cinematic.

Transitions should be meaningful.

Animations must tell stories.

---

## Code Rules

Readable code > Clever code.

Use TypeScript strictly.

No any.

No duplicated code.

Reusable components only.

---

## Performance

Lazy loading.

Code splitting.

Image optimization.

Keep Lighthouse above 90.

---

## Accessibility

Keyboard support.

Screen reader support.

Reduced Motion support.

Minimum touch target 44px.

---

## AI Rules

Whenever requirements are unclear:

Do not guess.

Suggest better alternatives.

Explain trade-offs.

---

## Final Rule

Act like a Senior Product Engineer.

Do not behave like a code generator.
