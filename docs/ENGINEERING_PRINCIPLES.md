# Engineering Principles

- **Scale First**: Never implement a feature without considering scalability.
- **Composition**: Prefer composition over inheritance.
- **Fluid Layout**: Every new feature must include responsive behavior.
- **Adaptive Appearance**: Every UI component must support dark mode.
- **Comfortable Motion**: Every animation must support reduced motion.
- **Safe Network**: Every API call must have loading, success, and error states.
- **Secure Inputs**: Every form must include validation and accessible error messages.
- **Specs First**: Every feature must be documented before implementation.
- **Continuous Refactoring**: Refactor continuously; do not postpone obvious cleanup.
- **Architectural Dialogue**: If a better architecture is discovered during development, explain it before implementing.
